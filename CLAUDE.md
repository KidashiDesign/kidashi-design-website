# Kidashi Design — Projektregeln

Code-Review-Standard: siehe `CODEREVIEW.md`

---

## Pflichtregeln

- PR-Beschreibungen und Commit-Messages enthalten **keine** AI-Zuschreibungen (`Generated by`, `Co-Authored-By: Claude` o.ä.)
- Branch-Namen: keine `claude/`-Prefix verwenden — stattdessen beschreibende Namen wie `feature/`, `fix/`, `update/`
- Keine persönlichen Daten der Inhaberin oder von Kunden in Commit-Messages, PR-Titeln oder Branches
- Esports-Projekt nur als „Freelance-Collaboration mit einem Esports-Event-Veranstalter" erwähnen
- Projekt Wiedmann & Winz: komplett aus Portfolio ausgeschlossen

## Deploy

FTP → Hostinger via GitHub Actions (`.github/workflows/deploy.yml`). Push auf `main` = Live-Deploy.

## Git-Workflow

Alle Änderungen werden direkt auf `main` committet und gepusht — es wird **nicht** auf separaten Feature-Branches gearbeitet, außer Nikki fragt explizit danach. Grund: Arbeit auf mehreren Branches gleichzeitig hat bereits dazu geführt, dass Änderungen verloren gingen bzw. nicht auf `main` landeten. Vor dem Push: lokalen Stand mit `origin/main` synchronisieren (fetch/pull), um Konflikte oder das versehentliche Mitziehen alter/unerwünschter Commits zu vermeiden.
