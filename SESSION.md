### 5. Code-Analyse: Galerie Kronsbein Phone-Mockup (auf Nutzeranfrage)
**Datei:** `portfolio/galerie-kronsbein/index.html` (Zeilen 51–76 Markup, 222–229 Inline-Script), `css/project.css` (Hero/Phone-Mockup-Regeln)

**Aufbau:**
- Zwei iframes in `.proj-hero__video-wrap`: Desktop-Hintergrund-Animation (`.proj-hero__video-frame.proj-hero__video-frame--desktop`, `src` direkt im HTML, `loading="eager"`) + Phone-Mockup (`#gkPhoneFrame.proj-hero__phone-iframe`, `data-src` statt `src`, `loading="lazy"`).
- CSS schaltet bei `≤600px` um: Desktop-Frame `display:none!important`, Phone-Mockup `display:flex`.
- Inline-Script am Ende von `<body>` prüft `matchMedia('(max-width:600px)')` **einmalig beim Laden** und setzt dann `gkPhoneFrame.src = dataset.src`.

**Gefundene Probleme:**
1. 🔴 **Race Condition:** `matchMedia`-Check läuft nur einmal, kein `resize`/`change`-Listener. Szenario: Seite wird auf Tablet im Querformat (>600px) geladen → Bedingung false → Phone-iframe bekommt nie einen `src`. Dreht der Nutzer das Gerät (<600px) → CSS zeigt das Mockup sofort an, aber der iframe bleibt leer → leere Hero-Section.
   - Fix-Vorschlag: `matchMedia(...).addEventListener('change', syncPhoneFrame)` ergänzen, plus Idempotenz-Check (`!el.src`).
2. 🟡 **Performance:** Die Desktop-Animation hat `src` fest im HTML (kein `data-src`) und `loading="eager"` — wird auf Mobile **immer geladen und ausgeführt** (nur per CSS versteckt, `display:none` verhindert kein Laden). Bei ~800 KB pro Animations-Bundle unnötiger Traffic/Akkuverbrauch auf Handys.
   - Fix-Vorschlag: gleiches `data-src`-Pattern wie beim Phone-iframe verwenden, per JS nur den jeweils sichtbaren iframe laden.
3. 🟢 Unkritisch: doppeltes `aria-hidden` (Wrapper + iframe), Breakpoint 600px konsistent zwischen JS und CSS.

**Bewertung: 6/10** — funktioniert im Normalfall, aber Lücke bei Resize/Rotation + unnötiger Mobile-Traffic.
**Noch nicht gefixt** — Nutzer wurde gefragt, ob die Fixes eingebaut werden sollen; Antwort stand bei Session-Ende noch aus.

## Projekt-Hero-Video/Animation — aktueller CSS-Stand (nach dieser Session)

```css
/* css/project.css */
.proj-hero--video .proj-hero__video-wrap {
  width: 100%;
  height: 100svh;      /* IMMER volle Viewport-Höhe, keine Mobile-Ausnahme mehr */
  overflow: hidden;
  display: block;
  position: relative;
}
.proj-hero--video .proj-hero__video-frame {
  position: absolute;
  top: 50%; left: 50%;
  width: max(100%, 177.78svh);   /* 16/9 als svh ausgedrückt: 100*16/9 */
  height: max(100svh, 56.25vw);  /* 9/16 als vw ausgedrückt: 100*9/16 */

  border: none; display: block; pointer-events: none;
}
```
